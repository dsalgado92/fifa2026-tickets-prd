## Story

**Como** aluno administrador da app FIFA 2026,
**eu quero** alimentar **resultados das partidas** pela área de admin (placar + encerrar partida),
**e ver a Tabela da Copa** atualizando automaticamente nos cards de cada grupo no site público,
**para que** a app fique interativa e didática — eu controlo o estado real do torneio inserindo placares, e a classificação reflete em tempo real conforme matches são encerrados.

---

## Contexto

Hoje a app tem:
- ✅ Schema `matches` com `home_score`, `away_score`, `status` (scheduled/live/finished/postponed)
- ✅ `PUT /api/matches/:id` aceita placar e status
- ✅ `AdminMatches.tsx` tem campos no form (mas UX de placar está enterrada no form completo de edição)
- ❌ Sem endpoint para classificação por grupo
- ❌ Sem página pública de tabela
- ❌ Sem item "Tabela" no Navbar

Esta story preenche os 3 gaps + adiciona um botão one-click no admin para encerrar partida (UX melhor para alimentar dados).

---

## Acceptance Criteria

### Backend

1. **AC-1** — Endpoint `GET /api/standings` retorna JSON `{ standings: { "A": [...], "B": [...], ..., "L": [...] } }`. Cada entrada por grupo é uma lista ordenada de objetos `{ team_id, team_name, team_code, team_flag, played, won, drawn, lost, gf, ga, gd, points }`.
2. **AC-2** — A agregação considera **somente** matches com `status = 'finished'` AND `home_score IS NOT NULL` AND `away_score IS NOT NULL`. Matches em outros estados não contam.
3. **AC-3** — Pontuação: vitória = 3 pts, empate = 1 pt, derrota = 0 pts. Cada team aparece com colunas calculadas: `played`, `won`, `drawn`, `lost`, `gf` (gols pró), `ga` (gols contra), `gd` (saldo = gf - ga), `points`.
4. **AC-4** — Ordenação dentro de cada grupo: `points` DESC → `gd` DESC → `gf` DESC → `team_name` ASC. Times sem jogos disputados aparecem por último com 0 em todas as colunas.

### Admin

5. **AC-5** — `AdminMatches.tsx` tem novo botão "Encerrar partida" (ícone `Flag` ou `CheckCircle`) por linha da tabela, que abre **modal compacto** com apenas dois campos numéricos (placar `home_score` e `away_score`) e dois botões: "Salvar e encerrar" (seta `status='finished'` + chama `PUT /api/matches/:id`) e "Cancelar".
6. **AC-6** — Após salvar, o modal fecha, lista de matches recarrega, e toast confirma "Partida encerrada com sucesso".
7. **AC-7** — Botão "Encerrar partida" só aparece em matches com `status != 'finished'` (matches já encerradas mostram apenas o placar com Badge "Encerrada"). Em matches encerradas, fica disponível um botão "Reabrir" que volta `status='scheduled'`.

### Público

8. **AC-8** — Nova rota `/standings` no `App.tsx` renderiza `Standings.tsx` (com Layout wrapper), seguindo o pattern visual das outras páginas (glass-card, gradient gold, header centralizado).
9. **AC-9** — `Standings.tsx` mostra **12 cards** (um por grupo A-L), cada um contendo uma tabela com colunas: `#` (posição), bandeira+nome, `J`, `V`, `E`, `D`, `GP`, `GC`, `SG`, `P`. Linhas ordenadas pela classificação. Top-2 destacados (background sutil ou ícone Trophy) — eles avançam.
10. **AC-10** — Item "Tabela" adicionado ao `Navbar.tsx` `navLinks` entre `'/groups'` e `'/qualified'`, com ícone `BarChart3` ou `Table`. Funciona em desktop e mobile menu.
11. **AC-11** — `lib/api.ts` ganha método `getStandings()` que chama `GET /api/standings`. `Standings.tsx` usa via `useQuery` com loading skeleton e error toast.

### Cross

12. **AC-12** — Smoke test ponta-a-ponta: encerrar uma partida via admin → recarregar `/standings` no público → tabela do grupo daquela partida reflete o resultado (pontos, gols).
13. **AC-13** — Build do frontend sem erros, backend `node -c` sem erros.

### 1. Backend — endpoint /api/standings (30 min)

- [ ] Criar `fifa2026-api/src/routes/standings.js`:
  ```js
  const express = require('express');
  const { query } = require('../config/database');

  const router = express.Router();

  // GET /api/standings
  router.get('/', async (req, res) => {
    try {
      // Buscar todos os teams em grupos (exclui Italy legacy com group_name=NULL)
      const teamsResult = await query(`
        SELECT id, name, code, flag, group_name
        FROM teams
        WHERE group_name IS NOT NULL
        ORDER BY group_name, name
      `);

      // Buscar matches finished com placar
      const matchesResult = await query(`
        SELECT id, home_team_id, away_team_id, group_name, home_score, away_score
        FROM matches
        WHERE status = 'finished'
          AND home_score IS NOT NULL
          AND away_score IS NOT NULL
          AND group_name IS NOT NULL
      `);

      const teams = teamsResult.recordset;
      const matches = matchesResult.recordset;

      // Inicializa standings por team
      const teamStats = new Map();
      for (const t of teams) {
        teamStats.set(t.id, {
          team_id: t.id,
          team_name: t.name,
          team_code: t.code,
          team_flag: t.flag,
          group_name: t.group_name,
          played: 0, won: 0, drawn: 0, lost: 0,
          gf: 0, ga: 0, gd: 0, points: 0,
        });
      }

      // Aplica resultados
      for (const m of matches) {
        const home = teamStats.get(m.home_team_id);
        const away = teamStats.get(m.away_team_id);
        if (!home || !away) continue; // skip se algum team não está em grupo

        home.played++; away.played++;
        home.gf += m.home_score; home.ga += m.away_score;
        away.gf += m.away_score; away.ga += m.home_score;

        if (m.home_score > m.away_score) {
          home.won++; home.points += 3;
          away.lost++;
        } else if (m.home_score < m.away_score) {
          away.won++; away.points += 3;
          home.lost++;
        } else {
          home.drawn++; home.points += 1;
          away.drawn++; away.points += 1;
        }
      }

      // Calcula gd
      for (const s of teamStats.values()) {
        s.gd = s.gf - s.ga;
      }

      // Agrupa por group_name
      const standings = {};
      for (const s of teamStats.values()) {
        if (!standings[s.group_name]) standings[s.group_name] = [];
        standings[s.group_name].push(s);
      }

      // Ordena cada grupo
      for (const g of Object.keys(standings)) {
        standings[g].sort((a, b) => {
          if (b.points !== a.points) return b.points - a.points;
          if (b.gd !== a.gd) return b.gd - a.gd;
          if (b.gf !== a.gf) return b.gf - a.gf;
          return a.team_name.localeCompare(b.team_name, 'pt-BR');
        });
        // Remove group_name de cada entry (já é a chave do dict)
        standings[g] = standings[g].map(({ group_name, ...rest }) => rest);
      }

      res.json({ standings });
    } catch (err) {
      console.error('Erro ao calcular tabela:', err);
      res.status(500).json({ error: 'Erro ao calcular tabela' });
    }
  });

  module.exports = router;
  ```

## Dev Notes

**Por que ordenação `localeCompare` em vez de simples `<`?**
PT-BR tem acentos. Para ordenar "Bélgica" antes de "Bósnia" corretamente, `localeCompare('pt-BR')` é o caminho.

**Por que Italy legacy (group_name=NULL) é skippado?**
Query do endpoint filtra `WHERE group_name IS NOT NULL`. Italy fica fora naturalmente.

**Por que destaca top-2 com Trophy ícone + bg sutil?**
Reforça visualmente a regra "2 primeiros avançam". Sem cor agressiva (verde, etc.) que quebraria o pattern do design.

**Padrão visual do site preserva-se:**
- `glass-card`, `gradient-gold`, `font-display` reutilizados
- Tabela `shadcn-ui` (não custom)
- Skeleton loading (mesmo padrão da Story 0.4 Dashboard admin)
- Toast de erro (mesmo padrão Story 0.5 TD-3)

**Por que admin tem botão "Reabrir"?**
Reduz fricção quando o aluno errar o placar — em vez de editar via form completo, clica "Reabrir" que volta status='scheduled', depois encerra de novo.

**Sobre cache do react-query:**
Após admin encerrar partida, a tabela em `/standings` só atualiza se o usuário recarregar OU se invalidarmos a query. Para evento, recarregar é suficiente. Em iteração futura, admin pode invalidar `['standings']` e `['matches']` via `queryClient.invalidateQueries()`.
